## 二、技术方案详解

### 2.1 SVG 曲线路径（穿针引线）

```
页面顶部
    │
    ├── M 550 50 (起点：正中央)
    │
    ├── C ... → 上古卡片右侧边缘 (x≈550, 针眼位置)
    │         └── 曲线弯向中间
    │              └── 弯向汉代卡片左侧边缘 (x≈548)
    │                   └── 曲线弯向中间
    │                        └── 弯向唐代卡片右侧边缘
    │                             └── ...循环...
    │                                  └── 终点
```

**实现方式：**
- `js/origin.js` → `generateCurvePath()` 遍历 `.era-card` 元素
- 读取每个卡片的 `getBoundingClientRect()` 计算实际位置
- 动态构建 SVG `<path d="...">` 使用三次贝塞尔曲线 (C)
- 每个卡片边缘设置 `.needle-eye`（22px 金色圆点，脉动动画）

### 2.2 滚动驱动曲线绘制

- 使用 `stroke-dasharray` + `stroke-dashoffset` 技术
- `initScrollDraw()` 监听 scroll 事件
- 计算 `.era-timeline-wrapper` 从进入视口到离开视口的滚动进度
- 进度从 0→1 映射到 dashoffset 从 totalLength→0
- 双层路径：底层 `thread-glow`（宽8px 低透明度光晕）+ 顶层 `thread-path`（2.5px 主线）

### 2.3 时代背景切换

6 个 body 类名对应 6 个时代：

| 类名 | 时代 | 背景特征 |
|------|------|---------|
| `era-ancient` | 上古 | 深蓝+原始祭火微暖光 |
| `era-han` | 汉代 | 深蓝+金色微光 |
| `era-tang` | 唐代 | 蓝紫+金色+紫色墨韵 |
| `era-song` | 宋代 | 深蓝+金色+暖橙+桂花黄 |
| `era-mingqing` | 明清 | 蓝+金色+红色+暗金 |
| `era-modern` | 当代 | 深蓝+流动金色多层光晕 |

使用 IntersectionObserver 检测当前可见卡片，动态切换 body 类名。背景过渡通过 CSS `transition: opacity 1.2s` 实现平滑切换。

### 2.4 卡片激活

- 每个 `.era-card` 默认 `opacity: 0.4; transform: translateY(20px)`
- 当卡片进入视口（IntersectionObserver threshold: 0.35），添加 `.active` 类
- `.active` → `opacity: 1; border-color: gold; box-shadow`
